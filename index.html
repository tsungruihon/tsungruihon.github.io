Support Vector Machine

发表于 2017-12-26 | 分类于 Machine Learning

这个算法一直困扰着我，虽然我没有在实际中使用过它，但是我早已听闻它的强大之处。但因为里面充满着各种数学推导，不花一些时间去研究实在是无法消化。但是，解决恐惧的最好方法就是直面它。我一直都在看各种相关书籍，以及Youtube上各种SVM的视频，可谓是受尽各种煎熬，当中可能会因为看不懂里面某个公式而坐立不安，然后又找不到人可以沟通请教。所以基本上这个SVM算法整天都在我的脑袋里面挥之不去，然后又没办法完全理清楚每一个细节，这样导致我没办法用代码复写一遍。整个人非常痛苦。但现在，我终于可以来写这个SVM分享文章了。

Support Vector Machine(SVM)，中文是支持向量机，是一种二分类模型。它的基本模型是定义在特征空间上的间隔最大的线性分类器。为什么要强调间隔最大？ 是因为能够将训练样本正确分开的划分超平面可能有很多，但是如何衡量哪一个是最好？间隔最大这一特性使SVM有别于感知机，因为间隔最大可以使对未见示例的泛化能力最强，而且保证对训练样本局部扰动的“容忍”性最好。

支持向量机主要分为以下三类线性可分支持向量机与硬间隔最大化、线性支持向量机与软间隔最大化以及非线形支持向量机与核函数。我一开始看这三个类别的时候，我心里简直就是想爆粗。不知道读者你们有没有类似的感受，如果你们没有，那么下文你们就不需要看了。对于跟我一样感受的读者，我们一起来看看到底这三类支持向量机到底在搞些什么毛线。

我们先来看看我们目前有什么。

在样本空间中，划分超平面可以通过以下线性方程来描述： \[ \mathbf{w}^{T}\mathbf{x} + b = 0 \] 其中， w=(w1; w2; ...; wd)为法向量，决定了超平面的方向；b为位移项，决定了超平面与原点之间的距离。然后样本空间中任意点x到超平面(w, b)的距离可写为： \[ r = \frac {|\mathbf{w}^{T}\mathbf{x} + b|}{||\mathbf{w}||} \] 假设超平面能够正确将训练样本分类，则有： \[ \left\{ \begin{array}{c} \mathbf{w}^{T}\mathbf{x} + b \ge +1, \ \ \ y_i=+1. \\ \mathbf{w}^{T}\mathbf{x} + b \le -1 \ \ \ y_i=-1.\\ \end{array} \right. \] 等价于 \[ y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) \ge +1 \] 距离超平面最近的几个训练样本能够使等号成立，它们被称为“支持向量(Support Vector)”，两个异类支持向量距离超平面的距离之和为： \[ \gamma= \frac{2}{||\mathbf{w}||} \]

一、线性可分支持向量机与硬间隔最大化

对于线性可分支持向量机，有一个很好的预期就是训练数据的分布没有任何冲突或者混杂，也就是说，这个训练数据集是线性可分。我们需要找出能够将两类数据正确划分并且间隔最大的直线。

构造并求解约束最优化问题： \[ \begin{aligned} & \mathop{min}_{w, b} \ \ \ \frac{1}{2}||w||^2 \\ \\ & s.t. \ \ \ y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) - 1\ge 0 \end{aligned} \] 然后构建拉格朗日函数: \[ L(w, b, \alpha) = \frac{1}{2}||w||^2 - \sum_{i=1} ^{N} \alpha_i\cdot((\mathbf{w}^{T}\mathbf{x} + b) - 1) \] 因此，问题变成极大极小问题： \[ \mathop{max}_\alpha\mathop{min}_{w, b} L(w, b, \alpha) \] （1）求min_{w, b}L(w, b, a) \[ \begin{aligned} \frac {\partial L}{\partial w} & = w - \sum_{i}^{N}\alpha_iy_ix_i = 0 \\ \\ \frac {\partial L}{\partial b} & = \sum_{i}^{N}\alpha_iy_i = 0 \end{aligned} \] 得 \[ \begin{aligned} & w = \sum_{i}^{N}\alpha_iy_ix_i \\ \\ & \sum_{i}^{N}\alpha_iy_i = 0 \end{aligned} \] 所以将这两个式子代入L(w, b, a)得： \[ \mathop{min}_{w, b} L(w, b, \alpha) = \sum_{i=1}^{N}\alpha_i - \frac {1}{2}\sum_{i=1}^{N}\sum_{j=1}^{N}\alpha_i\alpha_jy_iy_j(x_i \cdot x_j) \] （2）求min_{w, b} L(w, b, a)对a的极大： \[ \mathop{max}_{\alpha} L(w, b, \alpha) = \sum_{i=1}^{N}\alpha_i - \frac {1}{2}\sum_{i=1}^{N}\sum_{j=1}^{N}\alpha_i\alpha_jy_iy_j(x_i \cdot x_j) \] 这个问题的对偶问题是： \[ \begin{aligned} & \mathop{min}_{\alpha} \ \ \ L(w, b, \alpha) =\frac {1}{2}\sum_{i=1}^{N}\sum_{j=1}^{N}\alpha_i\alpha_jy_iy_j(x_i \cdot x_j) - \sum_{i=1}^{N}\alpha_i \\ & s.t \ \ \ \sum_{i=1}^{N}\alpha_iy_i = 0 \\ \\ & \ \ \ \ \ \ \ \ \ \alpha_i \ge 0 \\ \\ & \ \ \ \ \ \ \ \ \ y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) - 1\ge 0 \\ \\ & \ \ \ \ \ \ \ \ \ \alpha_i(y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) - 1) = 0 \end{aligned} \] 求得最优解a*=(a1*, a2*, ..., aN*)^T

（3）计算 \[ w^{*} = \sum_{i=1}^{N}\alpha_iy_ix_i \] 并选择a*的一个正分量aj*>0，计算得 \[ b^{*} = y_j - \sum_{i=1}^{N}\alpha_{j}^{*}y_i(x_i \cdot x_j) \] （4）求得分离超平面 \[ w^{*} \cdot x + b^{*} = 0 \] （5）分类决策函数 \[ f(x) = sign(w^{*} \cdot x + b^{*}) \] 解决了。

在线性可分支持向量机中，w*和b*只依赖于训练数据中对应aj*>0的样本点(xi, yi)，而其他样本点对w*和b*没有影响。因此我们把训练数据中对应aj*>0的实例点称为支持向量。

二、线性支持向量机与软间隔最大化

但是训练数据不一定总是线性可分，有时候某些样本点不一定满足间隔大于等于1得约束条件。因此，我们需要修改硬间隔最大化为软间隔最大化。也就是说，我们希望在最大化间隔的同时，不满足约束条件的样本应尽可能少。

于是优化目标可写为 \[ \mathop{min}_{w, b} \frac{1}{2}||w||^2 + C\sum_{i=1}^{m}\ell_{0/1}(y_i(\mathbf{w}^{T}\mathbf{x} + b )-1) \] 其中C>0使一个常数，𝓁0/1是“0/1损失函数” \[ \ell_{0/1}{(z)}= \begin{cases} 1, & \text{if $z$ <0;} \\[2ex] 0, & \text{otherwise.} \end{cases} \] 若C为无穷大时，这样会迫使所有样本都要满足原始约束。当C取有限值时，则允许一些样本不满足约束。

但𝓁0/1非凸、非连续，数学性质不太好，使得不易直接求解。于是常用其他替代损失函数来代替𝓁0/1。它们通常是凸的连续函数且是𝓁0/1的上界。 \[ \begin{aligned} & hinge \ \ loss: \ell_{hinge}(z) = max(0, 1-z) \\ \\ & exponential \ \ loss: \ell_{exp}(z) = exp(-z) \\ \\ & logistic \ \ loss: \ell_{log}(z) = log(1 + exp(-z)) \end{aligned} \]

引入松弛变量(slack variables)， 对每个样本点(xi, yi)引进一个松弛变量𝜉i>=0，使函数间隔加上松弛变量大于等于1。这时候，约束条件变为： \[ y_i(w \cdot x_i + b ) + \xi_i \ge 1 \]

同时，对每个松弛变量𝜉i， 支付一个代价𝜉i。这时候目标函数变成： \[ \frac {1}{2} ||w||^2 + C \sum_{i=1}^{N}\xi_{i} \] 这里，C>0称为惩罚参数，一般由应用问题决定。C值大时对误分类的惩罚增大，C值小时对误分类的惩罚减少。目标函数有两层含义，第一层是希望间隔尽可能大，同时使误分类点的个数尽量小，C是调和二者的系数。

构造并求解约束最优化问题： \[ \begin{aligned} & \mathop{min}_{w, b, 𝜉}\ \ \frac {1}{2} ||w||^2 + C \sum_{i=1}^{N}\xi_{i} \\ \\ & s.t. \ \ y_i(w \cdot x_i + b ) + \xi_i \ge 1 \\ \\ & \xi_i \ge 0, \ \ i=1, 2, ..., m. \end{aligned} \] 然后构建拉格朗日函数： \[ \begin{aligned} L(\mathbf{w}, b, \mathbf{\alpha}, \xi, \mu) = \mathop{min}_{w, b, 𝜉}\ \ & \frac {1}{2} ||w||^2 + C \sum_{i=1}^{N}\xi_{i} \\ & +\sum_{i=1}^{m}\alpha_i(1- \xi_i - (\mathbf{w}^{T}\mathbf{x} + b )-1)) - \sum_{i=1}^{m}\mu_i\xi_i \end{aligned} \] 其中a>=0，ui>=0是拉格朗日乘子。

（1）令L(w, b, a, 𝜉, u)对w, b, 𝜉的偏导为零可得： \[ \begin{aligned} \mathbf{w} & = \sum_{i=1}^{N}\alpha_iy_ix_i, \\ \\ 0 &= \sum_{i=0}^{m} \alpha_iy_i \\ C & = \alpha_i + \mu_i \end{aligned} \] （2）求min{w, b, 𝜉} L(w, b, a, 𝜉, u)对a的极大： \[ \begin{aligned} & \mathop{min}_{\alpha} \ \ \ \frac {1}{2}\sum_{i=1}^{m}\sum_{j=1}^{m}\alpha_i\alpha_jy_iy_j(x_i \cdot x_j) - \sum_{i=1}^{N}\alpha_i \\ & s.t \ \ \ \sum_{i=1}^{m}\alpha_iy_i = 0\\ \\ & \ \ \ \ \ \ \ \ \ 0 \le \alpha_i \le C , \ \ \ \mu_i \ge 0, \\ \\ & \ \ \ \ \ \ \ \ \ y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) - 1 + \xi_i \ge 0 \\ \\ & \ \ \ \ \ \ \ \ \ \alpha_i(y_i\cdot(\mathbf{w}^{T}\mathbf{x} + b) - 1+ \xi_i ) = 0 \\ \\ & \ \ \ \ \ \ \ \ \ \xi_i \ge 0, \ \ \ \mu_i\xi_i = 0 \end{aligned} \] （3）计算 \[ w^{*} = \sum_{i=1}^{N}\alpha_iy_ix_i \] 并选择a*的一个正分量0<=aj*<=C，计算得 \[ b^{*} = y_j - \sum_{i=1}^{N}\alpha_{j}^{*}y_i(x_i \cdot x_j) \] （4）求得分离超平面 \[ w^{*} \cdot x + b^{*} = 0 \] （5）分类决策函数 \[ f(x) = sign(w^{*} \cdot x + b^{*}) \] 解决了。

三、非线形支持向量机与核函数

对于线性分类问题，线性分类支持向量机是一种非常有效的方法，但是，有时候分类问题是非线性的。

这时候，想用一条直线去把这两类数据进行正确分类是不现实的。因此我们可以使用非线性支持向量机。

非线性支持向量机首先使用一个变换将原空间的数据映射到新空间；然后在新空间里用线性分类学习方法从训练数据中学习分类模型。实现这个过程最主要的方法就是核函数(Kernel Function)。

在原属性空间中计算的相似度函数K称为核函数(Kernel Function)，在满足Mercer原理，也就是说必须存在一个相应的变换，使得计算一对向量的核函数等价于在变换后的空间中计算这对向量的点积，能够使计算点积的开销更小，而且能够避免维灾难。也就是说，有 \[ \mathbf{K}(\mathbf{u, \mathbf{v}}) = \mathbf{\Phi}{(\mathbf{u})} \cdot \mathbf{\Phi}{(\mathbf{v})} = (\mathbf{u} \cdot \mathbf{v} + 1) ^ 2 \] 下面是一些常用的核函数

  • 多项式核函数（polynomial kernel function)

\[ K(x, z) = (x \cdot z + 1)^p \]

对应的支持向量机是一个p次多项式分类器。

  • 高斯核函数(Gaussian kernel function) \[ K(x, z) = exp\left(-\frac{||x-z||^2}{2\sigma^2}\right) \] 然后下面我们就可以尝试求解最优解了。

（1）选取适当的核函数K(x, z)和适当的参数C，构造并求解最优化问题 \[ \begin{aligned} & \mathop{min}_{\alpha} \ \ \ \frac {1}{2}\sum_{i=1}^{m}\sum_{j=1}^{m}\alpha_i\alpha_jy_iy_j(K(x_i, x_j)) - \sum_{i=1}^{N}\alpha_i \\ & s.t \ \ \ \sum_{i=1}^{m}\alpha_iy_i = 0\\ \\ & \ \ \ \ \ \ \ \ \ 0 \le \alpha_i \le C , \ \ \ \ i =1, 2, ..., m \\ \end{aligned} \] 求得最优解ɑ*=(ɑ1*, a2*, ..., a*N)^T

（2）选择ɑ*的一个正分量0<ɑj*<C，计算 \[ b^* = y_j - \sum_{i=1}^{N}\alpha_{i}^{*}y_iK(x_i \cdot x_j) \] （3）求得分离超平面 \[ \sum_{i=1}^{N}\alpha_{i}^{*}y_iK(x \cdot x_i) + b^{*} = 0 \] （4）分类决策函数 \[ f(x) = sign \left (\sum_{i=1}^{N}\alpha_{i}^{*}y_iK(x \cdot x_i) + b^{*}\right) \] 解决了。

Naive Bayes

发表于 2017-12-18 | 分类于 Machine Learning

总结完Logistic Regression，我觉得再看回Naive Bayes就很好理解了。从来都没有试过这么清楚过。

Naive Bayes(朴素贝叶斯法)是基于贝叶斯定理与特征条件独立假设的分类方法，对于给定的训练数据集，首先基于特征条件独立假设学习输入/输出的联合概率分布；然后基于此模型，对给定的输入x，利用贝叶斯定理求出后验概率最大的输出y。

朴素贝叶斯对条件概率分布作了条件独立性的假设，具体地，条件独立性假设是： \[ \begin{aligned} P(X =x|Y=c_k) &= P(X^{(1)} = x^{(1)}, ･･･, X^{(n)}=x^{(n)}|Y=c_k) \\ \\ & = \prod_{j=1}^nP(X^{(j)} = x^{(j)}|Y=c_k) \end{aligned} \] 独立条件假设就是说用于分类的特征在类确定的条件下都是条件独立的，这个公式就是似然函数。代表的是给定参数后变量的概率。这一假设使朴素贝叶斯变得简单，但代价就是有时候会牺牲一定的分类准确率。

朴素贝叶斯分类时，对给定的输入x，通过学习到的模型计算后验概率分布P(Y=ck|X=x)，将后验概率最大的类作为x的类输出。而后验概率计算是根据贝叶斯定理进行的： \[ \begin{aligned} P(Y=c_k|X=x) & = \frac {P(X=x|Y=c_k)\cdot P(Y=c_k)}{\sum_kP(X=x|Y=c_k)\cdot P(Y=c_k)} \\ \\ & =\frac {P(Y=c_k)\cdot \prod_jP(X^{(j)} |Y=c_k)}{\sum_kP(Y=c_k)\cdot \prod_jP(X^{(j)} |Y=c_k)} \ \ \ k=1, 2, ...,K \end{aligned} \] 于是朴素贝叶斯分类器可以表示为： \[ y = f(x) = \mathop{\arg\max}_{c_k} \frac {P(Y=c_k)\prod_jP(X^{(j)}=x^{(j)}|Y=c_k)}{\sum_kP(Y=c_k)\cdot \prod_jP(X^{(j)} |Y=c_k)} \] 然后其实对于每个c_k，分母都是一样的，所以可以简化公式： \[ y = f(x) = \mathop{\arg\max}_{c_k}P(Y=c_k)\prod_jP(X^{(j)}=x^{(j)}|Y=c_k) \] 搞定。

也就说，朴素贝叶斯法的学习与分类分为以下几个步骤：

  1. 计算先验概率及条件概率：

\[ \begin{aligned} & P(Y=c_k) = \frac {\sum_{i=k}^NI(y_i=c_k)}{N}, \ \ \ k=1, 2, ..., K \\ &P(X^{(j)} = a_{jl}|Y=c_k) = \frac {\sum_{i=1}^NI(x_i^{(j)} = a_{jl}, y=c_k)}{\sum_{i=1}^NI(y_i=c_k)} \\ \\ & j = 1, 2, ..., n; \ \ \ l = 1, 2, ..., S_j; \ \ \ k = 1, 2, ..., K \end{aligned} \]

  1. 对于给定的实例，可以进行计算：

\[ \begin{aligned} & x =(x^{(1)}, x^{(2)}, x^{(3)}, ..., x^{(n)})^T \\ \\ &P(Y=c_k)\prod_{j=1}^nP(X^{(j)} = x^{j}|Y = c_k), \ \ \ k = 1, 2, ..., K \end{aligned} \]

  1. 确定实例x的类：

\[ y = f(x) = \mathop{\arg\max}_{c_k}P(Y=c_k)\prod_jP(X^{(j)}=x^{(j)}|Y=c_k) \]

 然后下面我们可以用Python来实现这个算法啦。

首先，原来Logistic Regression的中文读法是逻辑斯谛回归而不是逻辑回归，知道真相的我忽然觉得以前的自己好土。。但这不是重点，其实这是重点。。好吧

Logistic Regression是根据不同的特征量，输出0和1，并且只能输出0和1。我们可以通过Sigmoid函数来实现。 \[ \sigma(x) = \frac{1}{1+e^{-w x}} = \frac{e^{wt}}{1 + e^{wx}} \]

\[ \sigma'(x) = \sigma(x)(1 - \sigma(x)) \]

而为什么会涉及到极大似然估计呢？因为我们希望能够在给定的概率密度函σ(x)，以及样本数据中，得到能够以最大概率发生样本数据的参数w的估计。因此我们需要引入似然函数并且计算极大似然估计。

确定了分类函数之后，接下来，我们把Sigmoid函数的输入记为z，则 \[ z = w_0x_0+w_1x_1+w_2x_2+...+w_nx_n = w^Tx \] 向量x是特征变量，是输入数据，向量w是回归系数。所以我们要做的事情就是如何确定最优的回归系数w(w0, w1, w2, ..., wn)

现在来看看我们有什么公式

  • 输入特征变量： \[z = w_0x_0+w_1x_1+w_2x_2+...+w_nx_n = w^Tx\]

  • 分类函数（预测函数）： \[ h(z) = \frac{1}{1+e^{-z}} = h_w(x)= \frac{1}{1+e^{-w^Tx}} \]

  • 对于任意确定的x和w，有：

    \[P(y=1|x, w)=h_w(x)\]

    \[P(y = 0|x, w) = 1 - h_w(x)\]

这个公式可以写成： \[ P(y|x, w) = (h_w(x))^y(1-h_w(x))^{1-y} \]

取似然函数： \[ \begin{aligned} L(w)& =\prod_{i=1}^{m}P(y^{(i)}|x^{(i)}, w) \\ & =\prod_{i = 1}^{m}(h_w(x^{(i)}))^{y^{(i)}}(1-h_w(x^{(i)}))^{1-y^{(i)}} \end{aligned} \] 然后转化为对数似然函数，这样更方便求解。 \[ J(w) = logL(w) = \sum_{i=1}^{m}(y^{(i)}logh_w(x^{(i)}) + (1-y^{(i)})log(1-h_w(x^{(i)}))) \] 得到对数似然函数J(w)后，我们就可以利用函数J(w)来表示预测值和实际值的偏差。J(w)越大，表示预测值与实际值的差异越小，预测越准确。

Cost函数可以写成： \[ Cost(h_w^x, y)=\left\{ \begin{aligned} logh_w(x), & & { y = 1} \\ 1 - logh_w(x), & & {y = 0} \end{aligned} \right. \] 下面我们需要求Cost函数的最大值，所以我们需要对J(w)进行求偏导： \[ \begin{aligned} \frac {\partial}{\partial {w_j}}J(w) & = \sum_{i=1}^m((y^{(i)}\frac{1}{h_w(x^{(i)})}-(1-y^{(i)})\frac{1}{1-h_w(x^{(i)})})\cdot\frac{\partial{h_w(x^{(i)})}}{\partial {w_j}}) \\ & = \sum_{i=1}^m((y^{(i)}\frac{1}{h_w(x^{(i)})}-(1-y^{(i)})\frac{1}{1-h_w(x^{(i)})})\cdot(h_w(x^{(i)})\cdot(1-h_w(x^{(i)}))\cdot x_j) \\ & = \sum_{i=1}^m((y^{(i)}\cdot (1-h_w(x^{(i)} )-(1-y^{(i)})\cdot h_w(x^{(i)})) \cdot x_j) \\ & = \sum_{i=1}^m(y^{(i)}-h_w(x^{(i)}))\cdot x_j) \end{aligned} \] 因此我们得到了梯度上升法迭代公式： \[ w_j := w_j + \alpha\sum_{i=1}^m(y^{(i)}-h_w(x^{(i)}))\cdot x_j) \] 弄清楚每一块做什么之后，我们就可以用Python写一遍Logistic Regression了。

而今天这一内容里面，我将分享自己对机器学习算法里面的梯度下降方法做一个笔记和总结。

在进行解释梯度下降方法之前，需要对方向导数和梯度做一个基本的解释。

对于方向导数，设 z = f(x, y) 在M0(x0, y0)处可微，则f(x, y)在M0点任意方向l = （cosa, cosb)存在方向导数且： \[ \frac{\partial f(x_0, y_0)}{\partial l} = \frac{\partial f(x_0, y_0)}{\partial x}cos\alpha+\frac{\partial f(x_0, y_0)}{\partial y}cos\beta \] 对于梯度，函数z = f(x, y)在点M0的方向导数计算公式可改写成：

在这里向量: \[ grad f(x_0, y_0) = (\frac{\partial f(x_0, y_0)}{x} , \frac{\partial f(x_0, y_0)}{\partial y} ） \]

称为z = f(x, y) 在点M0的梯度（向量）。

假设我们需要求得函数中某点变化最快的值，其实也就是求方向导数模的最大值，那么梯度的方向需要跟单位方向向量的夹角为0。也就是说，函数在某点的梯度，其方向是函数在该点的方向导数最大的方向，它的模是最大方向导数的值。

因此对于方向导数而言，求的是函数中某点在某个方向下的导数，不同方向的导数都不相同，也就说函数中某点沿不同方向的变化速度不一样。而梯度就是函数对变量偏导数的集合。

假设有如下一个线性回归问题，公式如下： > \[ > h(x) =h_\theta(x) =\theta_0+\theta_1x_1+\theta_2x_2 > \] >

其中x是自变量，比如说房子面积，θ是权重参数，也就是我们需要使用梯度下降求解的具体值。

然后我们还需要引入损失函数（Loss Function或者叫Cost Function)，目的是为了在梯度下降时用来衡量更新后的参数是否是向着正确的方向前进。如下损失函数(m表示训练集样本数量)

因为我们的目的，是要找到这个损失函数的值最小的参数下的目标函数h(x)，所以面对这种情况，我们其实很容易想到偏导数。结合方向导数以及梯度这一个概念，我们可以得到某个样本数据在函数中下降速度最快的方向。也就是说，我们希望能够对Loss Function求最优解，而我们可以通过求Loss Function的梯度然后不断更新参数来得到。如果我们得到了Loss Function的最优解，那么我们也就能得到目标函数h(x)。

import numpy as np


import matplotlib.pyplot as plt


#y=2 * (x1) + (x2) + 3 


rate = 0.001


X = np.array([[1, 2], [2, 1], [2, 3], 


              [3, 5], [1, 3], [4, 2],

              14, 8, 13, 


              20, 16, 28])




def compute_cost(X, y, theta):


    return np.sum(np.square(np.dot(X, theta) - y)) / (2 * len(y))




def gradient_descent_vectorization(X, y, learning_rate, iterations):


    X = np.column_stack((np.ones(len(X)), X))


    theta = np.zeros(X.shape[1])


    m = len(X)


    for i in range(iterations):


        hypothesis = np.dot(X, theta)


        loss = hypothesis - y


        gradient = np.dot(X.T, loss) / m


        theta = theta - learning_rate * gradient


    return X, theta




learning_rate = 0.01


iterations = 50000


X, theta = gradient_descent_vectorization(X, y, learning_rate, iterations)


cost = compute_cost(X, y, theta)




print("theta1 = ",theta[1], "theta2 = ", theta[2], "theta3 = ", theta[0])


print('cost:', cost)

x2 = [x]*3


x2

结果是[x]*3 生成一个列表x2，然后这个列表x2里面包括三个对象，每个对象都是列表x

所以当执行zip(*[x]*3)的时候，它首先unpack了x2这一个列表，把x2里面的对象全部拆分出来。所以这一行代码的步骤应该是以下：

  • [x]*3 ==> [x, x, x], 这里我令 buff = [x] * 3
  • *buff ==> x, x, x
  • 所以zip(*buff) ==> zip(x, x, x)

下面来看看，如果结合了迭代器会怎么样。例子： zip(*([iter(x)] * 2))

对于 zip(*([iter(x)] * 2)) ， 如果将 buff = iter(x)，那么将等同于 zip(buff, buff) 因此

zip(*([iter(x)]*2))




[output]


[([1, 2], [3, 4])]

因为iter()是迭代器，这里迭代了两次，第一次产出是[1, 2]， 第二次是[3, 4]。然后会将这次迭代结果组合成一个元组。而由于迭代两次之后，x里面已经没有对象可以进行迭代，所以返回这两个对象所组合的元组。

buff = iter(x)


zip(buff, buff)




[output]


[([1, 2], [3, 4])]

好了，解释完之后，那么tuples = zip(*([iter(flat)] * c))就很好理解了。

假设flat = [1, 2, 3, 4]，c = 2。

  1. 进行第一次总的迭代，里面分别迭代两次iter(flat), iter(flat)，将两次结果组合成元组，结果是(1,2)
  2. 因为flat里面还有两个元素3和4，恰好能凑合成两次迭代，所以再进行迭代，将两次结果组合成元组，返回结果(3, 4)
  3. 然后再将这两个结果组合成列表形式返回，结果是[(1, 2), (3, 4)]

假设flat = [1, 2, 3, 4]，c = 1。

  1. 进行第一次迭代iter(flat)，将结果组合成元组，结果是(1,)
  2. 进行第二次迭代iter(flat)，将结果组合成元组，结果是(2,)
  3. 进行第三次迭代iter(flat)，将结果组合成元组，结果是(3,)
  4. 进行第四次迭代iter(flat)，将结果组合成元组，结果是(4,)
  5. 将这四次迭代的结果组合成列表形式返回，结果是[(1,), (2,), (3, ), (4,)]

3.归一化（Normalization）

数据归一化用于对特征向量的值进行调整，保证每个特征向量的值都缩放到相同的数值范围。确保数据点没有因为特征的基本性质而产生较大差异，确保数据处于同一数量级，提高不同特征数据的可比性。

常用的归一化形式就是将特征向量调整为L1范数，使特征向量的数值之和为1。L0范数是指向量中非0的元素个数。如果用L0范数来规则化一个参数矩阵W，会希望W大部分元素都是0。L1范数和L2范数都可以实现矩阵稀疏，L1因具有比L0更好的优化求解特征而被广泛应用。而参数稀疏能够更好的进行特征选择以及增强解释性。L2范数在回归里面也叫“岭回归(Ridge Regression)”，也被称为权值衰减weight decay。岭回归能够改善模型过拟合问题。在L2范数中，限制了参数的变化波动范围，降低了标准差。总的而言，L2范数可以防止过拟合，提升模型的泛化能力。

解释器模式

在解释器模式中，有一个十分重要的名词，就是DSL(Domain Specific Language)，领域特定语言。DSL是一种针对一个特定领域的有限表达能力。

而DSL分为内部DSL和外部DSL。内部DSL构建在一种宿主编程语言中。内部DSL的一个例子是，使用Python解决线性方程组的一种语言。使用内部DSL的优势是我们不必担心创建、编译及解析语法，因为这些已经被宿主语言解决掉了。劣势是会受限于宿主语言的特性。如果宿主语言不具备这些特性，构建一种表达能力强、简洁而且优美的内部DSL是富有挑战性的。

享元模式

一个享元（Flyweight）就是一个包含状态独立的不可变（又称固有的）数据的共享对象，享元旨在优化性能和内存使用。若想享元模式有效，需要满足以下几个条件：

  • 应用需要使用大量的对象
  • 对象太多，存储／渲染它们的代价太大。一旦移除对象中的可变状态（因为在需要之时，应该由客户端代码显式地传递给享元），多组不同的对象可被相对更少的共享对象所替代。
  • 对象ID对于应用不重要。共享对象会造成ID比较的失败，所以不能依赖对象ID（那些在客户端代码看来不同的对象，最终具有相同的ID）。

一般来说，在应用需要创建大量的计算代价大但共享许多属性的对象时，可以使用享元。

模型-视图-控制器模式

关注点分离（Seperation of Concerns，SoC）原则是软件工程相关的设计原则之一。SoC原则背后的思想是将一个应用切分成不同的部分，每个部分解决一个单独的关注点。分层设计中的层次（数据访问层、业务逻辑层和表示层等）即是关注点的例子。使用SoC原则能简化软件应用的开发和维护。

一开始接触MVC模式是因为学习Django，Django把逻辑层、数据层和表示层都分开三个部分。所以各个模块的实现需要在对应的部分中填写，这样层次比较分明，减少了把所有部分都揉在一起而造成错误的问题。

代理模式

代理模式有以下四种：

  • 远程代理：实际存在于不同地址空间（例如，某个网络服务器）的对象在本地的代理者。
  • 虚拟代理：用于懒初始化，将一个大计算量对象的创建延迟到真正需要的时候进行。
  • 保护／防护代理：控制对敏感对象的访问。
  • 智能（引用）代理：在对象被访问时执行额外的动作。此类代理的例子包括引用计数和线程安全检查。

下面的例子是实现一个简单的保护代理来查看和添加用户。

class SensitiveInfo:


    def __init__(self):


        self.users = ['nick', 'tom', 'ben', 'mike']


        


    def read(self):


        print('There are {} users: {}'.format(len(self.users), ''.join(self.users)))


        


    def add(self, user):


        self.users.append(user)


        print('Added user {}'.format(user))


        


class Info:


    """SensitiveInfo的保护代理"""


    def __init__(self):

责任链模式

责任链（Chain of Responsibility）模式用于让多个对象来处理单个请求时，或者用于预先不知道应该由哪个对象（来自某个对象链）来处理某个特定请求时。在无法预知知道处理程序的数量和类型时，该模式有助于对请求／处理事情进行建模。适合使用责任链模式的系统例子包括基于事件的系统、采购系统和运输系统。

下面是用责任链模式实现一个简单的事件系统。

class Event:


    def __init__(self, name):


        self.name = name


    


    def __str__(self):


        return self.name


    


class Widget:


    def __init__(self, parent=None):


        self.parent = parent


    


    def handle(self, event):


        handler = 'handler_{}'.format(event)


        if hasattr(self, handler):


            method = getattr(self, handler)


            method(event)


        elif self.parent:


            self.parent.handler(event)


        elif hasattr(self, 'handler_default'):


            self.handler_default(event)




class MainWindow(Widget):


    def handle_close(self, event):


        print('MainWindow: {}'.format(event))


    


    def handler_default(self, event):


        print('MainWindow: {}'.format(event))


        


class SendDialog(Widget):


    def handle_paint(self, event):


        print('SendDialog: {}'.format(event))


        


class MsgText(Widget):


    def handler_down(self, event):


        print('MsgText: {}'.format(event))


        


def main():


    mw = MainWindow()


    sd = SendDialog(mw)


    msg = MsgText(sd)


    


    for e in ('down', 'paint', 'unhandled', 'close'):


        evt = Event(e)


        print('\nSending event -{}- to MainWindow'.format(evt))


        mw.handle(evt)


        print('Sending event -{}- to SendDialog'.format(evt))


        sd.handle(evt)


        print('Sending event -{}- to MsgText'.format(evt))


        msg.handle(evt)




if __name__ == '__main__':


    main()
