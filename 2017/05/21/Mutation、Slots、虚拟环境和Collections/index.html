Mutation、Slots、虚拟环境和Collections

发表于 2017-05-21 | 分类于 Python_Notes

这周继续阅读Python tips来了解Python知识，这周主要要深入了解的是Mutation、__slots__、虚拟环境(Virtualenv)和Collections。下面将分享我的阅读笔记。

一、Mutation

首先我们来看一个例子。

>>> foo = ['hi']
>>> print(foo)
['hi']
>>> bar = foo
>>> bar +=['bye']
>>> print(foo)
['hi', 'bye']

从上我们可以看出，每当将一个变量赋值为另一个可变类型的变量时，对这个数据的任意改动会同时反映到这两个变量上去。针对可变数据类型，新变量只不过是老变量的一个别名而已。下面我们来看一个列表的例子。

>>> def add_to(num, target=[]):
...    target.append(num)
...    return target
>>> add_to(1)
[1]
>>> add_to(2)
[1, 2]
>>> add_to(3)
[1, 2, 3]

上面这个例子，是因为列表是可变类型的数据结构。在Python中当函数被定义时，默认参数只会运算一次，而不是每次被调用时都会重新运算。我们应该尽量避免将可变类型参数定义为默认参数。如果我们想上面的例子每次重新运算。那我们可以这样做

>>> def add_to(element, target=None):
...    if target is None:
...        target = []
...    target.append(element)
...    return target
...
>>> add_to(1)
[1]
>>> add_to(2)
[2]

每次调用函数时，函数都没有传入target参数，因此一个新的列表会被创建。

二、__slots__

在Python中，每个类都有实例属性。默认情况下Python用一个字典来保存一个对象的实例属性。它允许我们在运行时去设置任意的新属性。

但是对于有着已知属性的小类来说，这个字典浪费了很多内存。Python不能在对象创建时直接分配一个固定量的内存来保存所有的属性。因此如果创建很多对象，则会消耗很多内存。而使用__slots__相当于告诉Python不要使用字典，而且只给一个固定集合的属性分配空间。

下面我们来看看在有与没有使用__slots__情况下的精确内存占用。

>>> import ipython_memory_usage.ipython_memory_usage as imu
>>> imu.start_watching_memory()
used 0.2383 MiB RAM in 6.24s, peaked 0.00 MiB above current, total RAM usage 41.67 MiB
>>> cd Desktop/python_exercise/
/Users/petertseng/Desktop/python_exercise
used 0.0586 MiB RAM in 0.10s, peaked 0.00 MiB above current, total RAM usage 41.73 MiB
>>> %cat slots.py
#!/usr/bin/env python
# encoding: utf-8

class MyClass(object):
    __slots__ = ['name', 'identifier']
    def __init__(self, name, identifier):
        self.name = name
        self.identifier = identifier

num = 1024 * 256
x = [MyClass(1,1) for i in range(num)]

used 0.0117 MiB RAM in 0.23s, peaked 0.00 MiB above current, total RAM usage 41.74 MiB
>>> %cat noslots.py
#!/usr/bin/env python
# encoding: utf-8

class MyClass(object):
    def __init__(self, name, identifier):
        self.name = name
        self.identifier = identifier

num = 1024 * 256
x = [MyClass(1,1) for i in range(num)]

used 0.0078 MiB RAM in 0.24s, peaked 0.00 MiB above current, total RAM usage 41.75 MiB
>>> from slots import *
used 24.5312 MiB RAM in 0.33s, peaked 0.00 MiB above current, total RAM usage 66.28 MiB
>>> from noslots import *
used 91.6484 MiB RAM in 0.45s, peaked 0.00 MiB above current, total RAM usage 157.93 MiB

由此可见，使用__slots__能够在满足已知小类的属性分配以及对内存的节省。

三、虚拟环境(Virtualenv)

虚拟环境是针对每个程序创建独立（隔离）的Python环境，而不是在全局安装所依赖的模块。

因为我用的是anaconda，所以在创建虚拟环境的步骤会不一样。

# 先更新conda
conda update conda

# 利用conda创建虚拟环境
conda create -n yourenvname python=x.x anaconda

# 激活虚拟环境
source activate yourenvname

# 安装额外的Python安装包
conda install -n yourenvname [package]

# 退出虚拟环境
source deactivate

# 删除虚拟环境
conda remove -n yourenvname -all

四、Collections

之前就有了解过Python的Collections，感觉这就是Python的一大神器。这里挑部分做总结分享。

defaultdict

正如之前笔记所记录，使用defaultdict不需要检查key是否存在，因为如果不存在，它会自动帮你加上。

>>> from collections import defaultdict
>>> colors = (
... ('Yasoob', 'Yellow'),
... ('Ali','Blue'),
... ('Arham','Green'),
... ('Ali','Black'),
... ('Yasoob','Red'),
... ('Ahmed','Silver'),
... )
...
>>> favourite_colors = defaultdict(list)
>>> for name, color in colors:
...    favourite_colors[name].append(color)
...    
>>> print(favourite_colors)
defaultdict(<type 'list'>, {'Arham': ['Green'], 'Yasoob': ['Yellow', 'Red'], 'Ahmed': ['Silver'], 'Ali': ['Blue', 'Black']})

而且，当我们需要在一个字典中对一个键进行嵌套赋值时候，使用defaultdict能绕过键不存在所引发的异常。

>>> import collections
>>> tree = lambda: collections.defaultdict(tree)
>>> some_dict = tree()
>>> some_dict['colors']['favourite'] = "Yellow"
>>> import json
>>> print(json.dumps(some_dict))
{"colors": {"favourite": "Yellow"}}

Counter

Counter是一个计数器，可以帮助我们针对某项数据进行计数。

>>> from collections import Counter
>>> colors = (
... ('Yasoob', 'Yellow'),
... ('Ali','Blue'),
... ('Arham','Green'),
... ('Ali','Black'),
... ('Yasoob','Red'),
... ('Ahmed','Silver'),
... )
...
>>> favs = Counter(name for name, color in colors)
>>> print(favs)
Counter({'Yasoob': 2, 'Ali': 2, 'Arham': 1, 'Ahmed': 1})

Deque

deque提供了一个双端队列，你可以从头／尾两端添加或删除元素。

>>> from collections import deque
>>> d = deque()
>>> d.append('1')
>>> d.append('2')
>>> print d
deque(['1', '2'])
>>> d.popleft()
'1'
>>> print d
deque(['2'])
>>> d.append('1')
>>> d.pop()
'1'

Nametuple

一个元组是一个不可变的列表，可以存储一个数据的序列。

>>> from collections import namedtuple
>>> Animal = namedtuple('Animal', 'name age type')
>>> perry = Animal(name='perry', age=31, type='cat')
>>> print(perry)
Animal(name='perry', age=31, type='cat')
>>> print(perry.name)
perry

我们可以用名字来访问nametuple中的数据，我们再继续分析它。一个命名元组nametuple有两个必需的参数。它们是元组名称和字段名称。

在上面的例子中，元组名称是Animal，字段名称是name，age和type。这样会让代码变得更容易维护。而且namedtuple的每个实例没有对象字典，所以它们很轻量，与普通的元组相比，并不需要更多的内存。所以它们比字典更快。

下面将一个命名元组转换为字典。

>>> from collections import namedtuple
>>> Animal = namedtuple('Animal', 'name age type')
>>> perry = Animal(name='perry', age=31, type='cat')
>>> print(perry._asdict())
OrderedDict([('name', 'perry'), ('age', 31), ('type', 'cat')])

OrderedDict

Python中默认的dict是无序的，但有些时候需要保持字典的有序性，这个时候可以使用OrderedDict，它是dict的一个subclass（子类），且保持了有序性。

# 以下是将无序的dict使用OrderedDict方法来让dict变得有序。
>>> from collections import OrderedDict
>>> d = {'Guava': 3, 'Persimmon': 1, 'Pitaya':4, 'Sugarcane':2}
# 将d按照key排序
>>> OrderedDict(sorted(d.items(), key=lambda t: t[0]))
OrderedDict([('Guava', 3), ('Persimmon', 1), ('Pitaya', 4), ('Sugarcane', 2)])
# 将d按照value排序
>>> OrderedDict(sorted(d.items(), key=lambda t: t[1]))
OrderedDict([('Persimmon', 1), ('Sugarcane', 2), ('Guava', 3), ('Pitaya', 4)])
# 将d按照key的长度排序
>>> OrderedDict(sorted(d.items(), key=lambda t: len(t[0])))
OrderedDict([('Guava', 3), ('Pitaya', 4), ('Sugarcane', 2), ('Persimmon', 1)])
# 使用popitem(last=True)方法可以使我们按照LIFO(Last in first out)的顺序删除dict中最后一个插入的键值对。
# 使用popitem(last=False)方法就按照FIFO(First in first out)的顺序删除dict中的键值对。
>>> d = {'Guava': 3, 'Persimmon': 1, 'Pitaya':4, 'Sugarcane':2}
>>> d = OrderedDict(sorted(d.items(), key=lambda t: t[0]))
>>> d
OrderedDict([('Guava', 3), ('Persimmon', 1), ('Pitaya', 4), ('Sugarcane', 2)])
>>> d.popitem() # 默认是popitem(last=True)
('Sugarcane', 2)
>>> d.popitem(last=False)
('Guava', 3)
