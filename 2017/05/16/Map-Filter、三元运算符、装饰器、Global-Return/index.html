Map&Filter、三元运算符、装饰器、Global&Return

发表于 2017-05-16 | 分类于 Python_Notes

上周继续了解Python知识里面的Map和Filter、三元运算符、装饰器、Global和Return。同样也是阅读了Python tips，下面将一步步分享我的笔记总结。

一、Map和Filter

Map函数作用是将一个函数映射到一个输入列表的所有元素上。

# 以下是它的一般表达式
map(function_to_apply, list_of_inputs)

# 把列表中所有元素一个个地传递给一个函数，并收集输出
>>> items = [1,2,3,4,5]
>>> squared = list(map(lambda x: x**2, items))
>>> print squared
[1, 4, 9, 16, 25]

# 用列表包含多个函数然后映射到列表元素上
>>> def multiply(x):
...    return (x*x)
...
>>> def add(x):
...    return (x+x)
...
>>> funcs = [multiply, add]
>>> for i in range(5):
...    value = list(map(lambda x: x(i), funcs))
...    print(value)
...    
[0, 0]
[1, 2]
[4, 4]
[9, 6]
[16, 8]

Filter函数能创建一个列表，其中返回列表里面的每个元素都能对函数返回True值。Filter函数类似于一个For循环，但是它是一个内置函数，并且更快。

>>> number_list = range(-5,5)
>>> less_than_zero = list(filter(lambda x:x <0, number_list))
>>> print(less_than_zero)
[-5, -4, -3, -2, -1]

二、三元运算符

三元运算符通常在Python里被称为条件表达式，这些表达式是基于真(True)/假(Not)的条件判断。下面是一个伪代码 condition_is_true if condition else condition_is_false

例子：

>>> is_fat = True
>>> state = "fat" if is_fat else "not fat"
>>> print state
fat

它允许用简单的一行快速判断，而不是使用复杂的多行if语句。这在大多数时候非常有用，而且可以使代码简单可维护。

三、装饰器

装饰器(Decorators)是Python的一个重要部分，它们是修改其他函数的功能函数。有助于让代码更短，更Pythonic。

1.1 一切皆对象

首先理解一下Python中的函数

# 首先定义一个函数
>>> def hi(name='yasoob'):
...    return "hi " + name
>>> print(hi())
hi yasoob
# 然后把一个函数赋值给一个变量
# 这里没有使用小括号hi()，因为并不是在调用hi函数
# 而是将它放在greet变量里，是一个深拷贝
>>> greet = hi
>>> print(greet())
hi yasoob
>>> del hi
>>> print(hi())
NameErrorTraceback (most recent call last)
<ipython-input-15-3486093e8057> in <module>()
----> 1 print(hi())

NameError: name 'hi' is not defined
# 因此删除了hi函数时候
# 仍能调用greet函数
>>> print(greet())
hi yasoob

1.2 在函数中定义函数

下面，我们来看看如何在一个函数中定义另外一个函数

>>> def hi(name="yasoob"):
...    print("now you are inside the hi() function")
...
...    def greet():
...        return "now you are in the greet() function"
...
...    def welcome():
...        return "now you are in the welcome() function"
...
...    print(greet())
...    print(welcome())
...    print("Now you are back in the hi() function")
    

>>> hi()
now you are inside the hi() function
now you are in the greet() function
now you are in the welcome() function
Now you are back in the hi() function

# 上面展示了无论什么时候调用hi()，greet()和welcome()将会同时被调用
# 然后greet()和welcome()函数在hi()函数之外是不能访问的
>>> welcome()

NameErrorTraceback (most recent call last)
<ipython-input-21-efaf77b113fd> in <module>()
----> 1 welcome()

NameError: name 'welcome' is not defined

现在我们知道了可以在函数中定义另外的函数，即创建嵌套的函数。

1.3 从函数中返回函数

我发现，想要更好的理解这个topic，应该要先学习一下Python函数加括号和不加括号的区别

>>> def l1():
...    return "hello"
>>> l1
... <function __main__.l1>
>>> l1()
... 'hello'

调用l1打印的是l1的内存地址，而且不加括号的话，函数可以用来被到处传递，并且可以赋值给别的变量而不去执行。而调用l1()是执行l1方法，返回hello

下面我们尝试在一个函数里面输出另外一个函数作为输出返回。

>>> def hi(name='yasoob'):
...    def greet():
...        return "Now you are in the greet() function"
...    def welcome():
...        return "Now you are in the welcome() function"
...    if name == "yasoob":
...        return greet
...    else:
...        return welcome
...
# 当我们赋予hi()给a时，hi()会被执行
# 而执行的结果，只是返回greet()函数本身
>>> a = hi()
# 下面显示了`a`现在是指向了hi()函数中的greet()函数
# 但并没有执行greet()函数
>>> print(a)
<function greet at 0x111406aa0>
>>> print(a())
Now you are in the greet() function
>>> print hi
<function hi at 0x111406140>
>>> print hi()
<function greet at 0x110a3a7d0>
# 下面这一步就显得非常直接了
# 第一个括号是执行hi()函数，指向greet()函数
# 第二个括号是执行greet()函数，并且返回对应的结果
>>> print hi()()
Now you are in the greet() function

1.4 将函数作为参数传给另一个函数

>>> def hi():
...    return "hi yasoob!"
...
# 将hi()函数作为参数传给doSomethingBeforeHi()函数
>>> def doSomethingBeforeHi(func):
...    print("I am doing some boring work before executing hi()")
...    print(func())
...
>>> doSomethingBeforeHi(hi)
I am doing some boring work before executing hi()
hi yasoob!

1.5 第一个装饰器

现在我们更改一下上一个程序，尝试写一个更有用的装饰器。

>>> def a_new_decorator(a_func):
...    
...    def wrapTheFunction():
...        print("I am doing some boring work before executing a_func()")` 
...        a_func()
...        print("I am doing some boring work after executing a_func()")
...    
...    return wrapTheFunction
...
>>> def a_function_requiring_decoration():
...    print("I am the function which needs some decoration to remove my soul smell") 
...
>>> a_function_requiring_decoration()
I am the function which needs some decoration to remove my soul smell
# 将a_function_requiring_decoration()函数作为参数传递给a_function_requiring_decoration()函数
>>> a_function_requiring_decoration = a_new_decorator(a_function_requiring_decoration)
# 调用a_function_requiring_decoration()函数
# 将会指向a_function_requiring_decoration地址
>>> a_function_requiring_decoration
<function __main__.wrapTheFunction>
# 执行a_function_requiring_decoration()函数
# 其中a_function_requiring_decoration()函数作为参数
# 将会返回wrapTheFunction()函数
>>> a_function_requiring_decoration()
I am doing some boring work before executing a_func()
I am the function which needs some decoration to remove my soul smell
I am doing some boring work after executing a_func()

因此，上面其实实现了封装函数，或者说已经实现了Python装饰器做的事情。只不过没有使用@符号。那么下面我们使用@符号来运行上一个函数例子。

>>> @a_new_decorator
... def a_function_requiring_decoration():
...    print("I am the function which needs some decoration to"
...    "remove my foul smell")    
...
>>> a_function_requiring_decoration()
I am doing some boring work before executing a_func()
I am the function which needs some decoration toremove my foul smell
I am doing some boring work after executing a_func()

# 但当我们运行下面一行代码的时候
# 会发现它并不是返回`a_function_requiring_decoration`
# 因为这里的函数被`wrapTheFunction`所替代
# 装饰器在这里重写了函数的名字和注释文档(docstring)
>>> print(a_function_requiring_decoration.__name__)
wrapTheFunction

# 幸运的是，我们可以使用functools.wraps
>>> from functools import wraps
>>> def a_new_decorator(a_func):
...    @wraps(a_func)
...    def wrapTheFunction():
...        print("Something before executing a_func()")
...        a_func()
...        print("Something after executing a_func()")
...    return wrapTheFunction
...
>>> @a_new_decorator
>>> def a_function_requiring_decoration():
...    print("I am the function which needs some decoration to"
...    "remove my foul smell")
...    
>>> print(a_function_requiring_decoration.__name__)
a_function_requiring_decoration

书中给了一个蓝本规范，这是我第一次真正明白装饰器，也是第一次感觉到装饰器的魅力之处。

>>> def decorator_name(f):
...    @wraps(f)
...    def decorated(*args, **kwargs):
...        if not can_run:
...            return "Function will not run"
...        return f(*args, **kwargs)
...    return decorated
...
>>> @decorator_name
>>> def func():
...    return("Function is running")
...
>>> can_run = True
>>> print(func())
Function is running
>>> can_run = False
>>> print(func())
Function will not run

其中值得注意的是，@wraps接受一个函数来进行装饰，并加入了函数名称、注释文档、参数列表等等的功能。这可以让我们在装饰器里面访问在装饰之前的函数的属性。

1.6 使用场景

下面将会介绍装饰器的使用场景，来看看它是如何让一些事情管理起来变得更简单。

授权

装饰器能够有助于检查某个人是否被授权去使用一个web应用的端点(endpoint)。它们被大量使用于Flask和Django这类web框架中。下面将展示一个例子。

>>> def requires_auth(f):
...    @wraps(f)
...    def decorated(*args, **kwargs):
...        auth = request.authorization
...        if not auth or not check_auth(auth.username, auth.password):
...            authenticate()
...        return f(*args, **kwargs)
...    return decorated

日志

>>> def logit(func):
...    @wraps(func)
...    def with_logging(*args, **kwargs):
...        print(func.__name__) + "was called"
...        return func(*args, **kwargs)
...    return with_logging
...    
>>> @logit
>>> def addition_func(x):
>>>    print x+x
...    
>>> result = addition_func(4)

在函数中嵌入装饰器

>>> def logit(logfile='/Users/petertseng/Desktop/out.log'):

>>> class logit(object):

Global&Return

在这一小节中，主要是了解Global和Return，首先我们来看一段代码

>>> def add(value1, value2):
...    return value1 + value2
...
>>> result = add(3, 5)
>>> print(result)
8

上面一个例子中，我们把值赋予给result。函数中一定要有return返回值，这样才是完整的函数。如果定义的函数没有返回值，那么会得到一个None的对象。

>>> def add(value1, value2):
...    print value1 + value2
>>> result = add(3,5)
8
>>> print result
None

同样我们也可以这样写

>>> def add(value1, value2):
...    global result
...    result = value1 + value2
...    
>>> add(3, 5)
>>> print(result)
8

而在第二个例子中，出现了global关键字。global关键字意味着可以在函数以外的区域都能访问这个变量。但是一般而言，我们应该尽量避免使用global关键字，这相当于引入了多余变量刀全局作用域。
