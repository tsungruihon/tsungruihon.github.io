二、对象自省

自省(Introspection)，在计算机编程领域里，是指在运行时来判断一个对象类型的能力，也就说它是什么、它跟神什么相干以及它能做什么。Python中所有一切都是对象，可以仔细勘察那些对象。Python中包含许多内置函数和模块来帮助我们。

二、Debugging

程序能一次写完并正常运行的概率很小，基本不超过1%。总会有各种各样的bug需要修正。有的bug很简单，看看错误信息就知道，有的bug很复杂，我们需要知道出错时，哪些变量的值是正确的，哪些变量的值是错误的，因此，需要一整套调试程序的手段来修复bug ——廖雪峰

在本节中，收获到最重要的信息是两个，一个是Debugging很重要，二是Python的Debugging中的pdb.set_trace()十分有用。

之后便进入了debugger模式，下面是debugger模式下的一些命令：

  • c: 继续执行
  • w: 显示当前正在执行的代码行的上下文信息
  • a: 打印当前函数的参数列表
  • s: 执行当前代码行，并停在第一个能停的地方（相当于单步进入，就是一行一行地执行代码）
  • n: 继续执行到当前函数的下一行，或者当前行直接返回（相当于单步跳过）

单步跳过（next）和单步进入（step）的区别在于，单步进入会进入当前行调用的函数内部并停在里面，而单步跳过会（几乎）全速执行完当前调用的函数，并停在当前函数的下一行。

三、Generators

本节在介绍生成器（Generators）的时候，引导读者先要理解迭代器（iterators）。

迭代器是一个让程序员可以遍历一个容器（特别是列表）的对象。然而，一个迭代器在遍历并读取一个容器的数据元素时，并不会执行一个迭代。

里面出现有三个核心部分：

  • 可迭代对象（Iterable）
  • 迭代器（Iterator）
  • 迭代（Iteration）

下面将分别讨论以上三个部分。

3.1 容器（container）

在介绍可迭代对象前，我找了相关资料，发现必须要先介绍Python中的容器（container）概念。

容器是一种把多个元素组织在一起的数据结构，容器中的元素可以逐个地迭代获取，可以用in、not in关键字判断元素是否包含在容器中。通常这类数据结构把所有的元素存储在内存中（也有一些特例，并不是所有的元素都在内存，比如迭代器和生成器对象）。

在Python中，常见的容器有：

  • list, deque, …
  • set, frozenset, …
  • dict, defaultdict, OrderedDict, Counter, …
  • tuple, namedtuple, …
  • str

上述当中，我对defaultdict, OrderedDict, Counter和namedtuple比较陌生，所以我在Python官方文档中做了一些资料查找阅读。

defaultdict

在使用Python原生的dict数据结构时，如果用d[key]访问但指定的key不存在时，是会抛出KeyError异常的。

但如果使用defaultdict，只要传入一个默认的工厂方法，那么请求访问一个不存在的key时，便会调用这个工厂方法使用其结果来作为这个key的默认值。

OrderedDict

Python中默认的dict是无序的，但有些时候需要保持字典的有序性，这个时候可以使用OrderedDict，它是dict的一个subclass（子类），且保持了有序性。

Counter

Counter也是dict的一个subclass，它是一个无序容器，可以看作一个计数器，用来统计相关元素出现的个数。(也是好美啊)

namedtuple

我对namedtuple的理解就是，用namedtuple来为元组每一个内容赋予一个特定的访问类别，它跟tuple的一大重要区别就是，namedtuple不仅能让代码变得更具有可读性，更轻量，而且相比于原生tuple所需占用的内存更少。

3.3 迭代器(iterator)

迭代器是一个带有状态的对象：

  • 在迭代中带有记得迭代到哪个成员的状态
  • 可以利用__next__方法：
    • 在迭代中返回下一个值
    • 当指向下一个值时会更新状态
    • 当结束迭代时会通过StopIteration来发起停止信号
  • 迭代器用__iter__方法返回的迭代器就是它自身本身

其中itertools函数返回的都是迭代器对象

为了更直接地感受迭代器内部的执行过程，以斐波那契数列为例定义一个迭代器。

Fib既是一个可迭代对象，因为它实现了__iter__方法。同时又是一个迭代器，因为它实现了next犯法。每次调用next()方法时候做两件事：

  1. 为下一次调用next()方法修改状态
  2. 为这次调用生成返回结果

迭代器就像一个懒加载的工厂，等到有人需要的时候才给它生成值返回，没调用的时候就处于休眠状态等待下一次调用。
